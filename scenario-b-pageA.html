This time, the attack happens differently — Page B redirects itself to Page C.

Attack Flow (Scenario B)
  1. You click the link — Page B opens in a new tab.
  2. You stay on this tab (or return to it).
  3. After a few seconds, Page B automatically redirects itself to Page C.
  4. You click back to the Page B tab — and it looks like a login page you trust.
The Link (Looks Safe)
<a href="scenario-b-pageB.html" target="_blank"> Visit Page B </a>

No rel="opener" — but older browsers may still expose window.opener without it. More importantly, the attacker simply redirects Page B itself after a timeout — no opener access needed.
